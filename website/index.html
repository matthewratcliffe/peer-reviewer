Built for real engineering teams

Not a linter. Not a formatter. A reviewer that finds the things automated tools miss.

🔍

Real bug detection

Catches logic errors, off-by-one mistakes, wrong conditionals, and incorrect assumptions that slip through unit tests.

🔒

Security & penetration testing

Identifies injection risks, auth gaps, privilege escalation paths, parameter tampering, and unsafe deserialization.

⚡

Performance awareness

Flags N+1 queries, unnecessary allocations in hot paths, and quadratic algorithms where linear ones are straightforward.

📋

Organisation coding standards

Point it at a folder of Markdown files with your team's standards. They become part of every review, automatically enforced.

🔀

Pre-commit gate

Optionally block commits when unresolved medium/high findings exist. Catches issues before they enter your branch.

🏠

Fully local & private

Runs entirely on your machine over a local IPC socket. Your code never leaves your workstation. No cloud, no telemetry.

Bring your own LLM

Virtual Peer Review uses locally-installed CLI tools or a local llama.cpp server. You need an existing subscription or local model — the tool itself is free.

⚠ Requires a working local CLI subscription (Claude, Codex, Kiro, OpenCode) or a running llama.cpp instance. No API keys are entered into Virtual Peer Review — it delegates to whatever auth the CLI already has.

Up and running in 2 minutes

1

Install the extension

Grab the extension for your IDE from GitHub Releases, or install the TUI via npm:
npm i -g virtualpeerreview

2

Have an LLM ready

Make sure you have Claude CLI, Codex CLI, Kiro CLI, OpenCode CLI, or a llama.cpp server running locally.

3

Open your project

The service starts automatically when your IDE opens. Findings appear as you save or run analysis.
